Retrieval → Pre-ranking → Ranking → Re-ranking の 4 段バトンリレーを、各段の損失関数・モデル構造・基盤技術まで踏み込んで解説する。

数百万〜数十億件のアイテムを、各段で「精度を上げながら件数を絞る」バトンリレー。 前段ほど 速度と再現率（Recall）、後段ほど 精度（Precision）と体験最適化 を優先する。 前段で誤って落とした候補は後段で復活できないため、Retrieval の再現率が全体性能の上限を決める。

全カタログ $N=10^8$ 件すべてに重い深層ランキングモデル（1件あたり数百μs）を適用すると、1リクエストで数万秒かかり非現実的。 そこで「軽くて速いが粗い」モデルで $N$ を $10^3$ 程度まで一気に落とし、「重くて正確」なモデルを少数の候補にだけ使う。 各段は「前段の出力分布」を入力とするため、段ごとに Sample Selection Bias（前段を通過したものしか学習で見えない）が生じる点が実務上の難所。

第1段 Retrieval（候補生成）

目的：数百万個のアイテムから、好みに合いそうな上位 100〜1,000 件を ミリ秒単位でくり抜く。 精密な順位は問わない。「正解を取りこぼさないこと（高 Recall）」が至上命題。

現代の標準は マルチチャネル（複数の網を併用）。1 つのモデルに頼らず、性質の違う候補生成器を並列で走らせ、結果をマージ・重複除去してから次段へ渡す。これにより、ある 1 チャネルの弱点（例：Two-Tower の人気バイアス、コンテンツベースのフィルターバブル）を他チャネルが補完する。

非パーソナライズ型

人気順・トレンド・急上昇。全ユーザー行動の集計。コールドスタート時の安全網。

コンテンツベース

アイテム属性／マルチモーダル埋め込みで「中身が似たもの」を探す。新着に強い。

協調フィルタリング

メモリベース（近傍法）／モデルベース（行列分解）。「似た行動の人／よく一緒に買われるペア」。

Two-Tower

user/item を同じ空間に射影し対照学習。ANN で高速取得。現代 Retrieval の主力。

グラフ（GNN）

相互作用をグラフ化し数ホップ先まで信号伝播。間接的つながりを捉える。

生成型リトリーバル

Semantic ID を Transformer で生成。ANN 不要で候補を「生成」する次世代手法。

協調フィルタリング（メモリベース）

数式モデルを学習せず、ユーザー×アイテムの生の行動行列から直接「似た行動」を計算する古典手法。Amazon 初期の Item-to-Item CF が代表例。

ユーザーベース / アイテムベース

  • ユーザーベース： 自分と行動が似たユーザー（近傍）を見つけ、その人たちが好きで自分が未見のアイテムを推薦。
  • アイテムベース： 「アイテム $i$ を見た人は $j$ も見る」という共起から、アイテム間類似度を事前計算。ユーザー数 ≫ アイテム数の EC で安定し、Amazon が採用して有名に。

類似度はコサイン類似度やピアソン相関を使う。ユーザー $u,v$ の評価ベクトル $r_u, r_v$ に対し:

$$\text{sim}(u,v)=\frac{\sum_{i\in I_{uv}}(r_{ui}-\bar r_u)(r_{vi}-\bar r_v)}{\sqrt{\sum_{i}(r_{ui}-\bar r_u)^2}\sqrt{\sum_{i}(r_{vi}-\bar r_v)^2}}$$
⚠ 弱点
スパース性（評価がほぼ空欄）とスケーラビリティ（ユーザー増で計算爆発）、そしてコールドスタート。これらを克服するために登場したのが次の 行列分解。

協調フィルタリング（行列分解：MF / SVD / ALS）

巨大でスパースな「ユーザー×アイテム」評価行列 $R \in \mathbb{R}^{m\times n}$ を、2 つの薄い 潜在因子行列 $P\in\mathbb{R}^{m\times k}$（ユーザー）と $Q\in\mathbb{R}^{n\times k}$（アイテム）の積で近似する。Netflix Prize（2009）で世界標準に。

$$\hat r_{ui}=p_u^\top q_i,\qquad \min_{P,Q}\sum_{(u,i)\in\mathcal{K}}\left(r_{ui}-p_u^\top q_i\right)^2+\lambda\left(\|p_u\|^2+\|q_i\|^2\right)$$
  • SVD / FunkSVD： SGD で観測済みエントリのみ最小化。バイアス項 $\mu+b_u+b_i$ を加えるのが定番。
  • ALS（Alternating Least Squares）： $P$ と $Q$ を交互に固定して閉形式で解く。並列化しやすく Spark MLlib などで大規模対応。暗黙的フィードバック（クリック有無）向けの WRMF が実用的。
  • BPR（Bayesian Personalized Ranking）： 「観測ペア $>$ 未観測ペア」のペアワイズ順位損失。暗黙的フィードバックの Top-N 推薦に強い。
▶ Two-Tower との関係
MF は「ユーザー ID とアイテム ID だけを特徴とする最も単純な Two-Tower」とみなせる。ここに任意の特徴量と非線形変換（NN）を入れて一般化したのが次の Two-Tower。

Two-Tower（双塔モデル / 対照学習）

ユーザー側（user/context 特徴）とアイテム側（item 特徴）を 別々のエンコーダ（塔）で処理し、同じ次元の埋め込み空間に射影。両者の内積／コサインを相性スコアとする。

$$s(u,i)=\frac{f_\theta(x_u)^\top g_\phi(x_i)}{\|f_\theta(x_u)\|\,\|g_\phi(x_i)\|}$$

対照学習（In-batch Negatives + InfoNCE）

正例（クリックした item）を近づけ、負例（その他）を遠ざける。バッチ内の他サンプルを負例に流用する in-batch negative が効率的:

$$\mathcal{L}=-\frac{1}{B}\sum_{u}\log\frac{\exp(s(u,i_u^+)/\tau)}{\sum_{j\in\text{batch}}\exp(s(u,i_j)/\tau)}$$
  • 人気バイアス補正： in-batch negative は人気アイテムが負例に出やすいため、logQ correction（サンプリング確率でロジット補正）が必須。Google の論文（Yi et al. 2019）が有名。
  • 推論時の決定的な利点： アイテム塔はオフラインで全アイテムを事前にベクトル化し ANN インデックスに格納できる。リクエスト時はユーザー塔だけ計算 → ANN 検索、で完結。
  • 進化： 塔の中身は初期の MLP から、系列を扱う SASRec / BERT4Rec（下記）へ。user-item に context を加えた Three-Tower も。
⚠ 構造的制約
user と item の interaction を 最後の内積でしか表現できない（late interaction）。これは ANN を可能にする代償。リッチな特徴量交差は後段の Ranking に任せる、という役割分担。

グラフベース（GNN）

ユーザー×アイテムの相互作用を 二部グラフとして捉え、エッジに沿って埋め込みを伝播（メッセージパッシング）。「自分が買った商品を買った人が買った別の商品」のような 高次の間接的つながりを捉えられる。

モデル特徴
PinSagePinterest。ランダムウォークで近傍サンプリング＋GraphSAGE。30 億ノードの本番グラフで動く工学的金字塔。
LightGCNGCN から特徴変換・非線形活性を撤廃し「近傍平均の伝播だけ」に簡素化。協調フィルタリング用途では精度↑＆軽量。
GraphSAGE近傍をサンプリングして集約。帰納的（未知ノードにも適用可）でインダクティブ。

LightGCN の伝播式（$l$ 層目の埋め込み $e^{(l)}$、$N$ は近傍）:

$$e_u^{(l+1)}=\sum_{i\in N_u}\frac{1}{\sqrt{|N_u|}\sqrt{|N_i|}}e_i^{(l)},\qquad e_u=\sum_{l=0}^{L}\alpha_l\,e_u^{(l)}$$

シーケンシャル / 自己回帰型

ユーザー行動を 時系列とみなし「次に何を見る／買うか」を予測。直近の興味の変化を強く反映でき、セッション推薦に必須。

  • SASRec： 単方向 Self-Attention（GPT 的）で「次アイテム」を自己回帰予測。長期・短期の依存を Attention の重みで動的に捉える。
  • BERT4Rec： 双方向 Transformer ＋ Masked Item Prediction（Cloze タスク）。系列の前後文脈を使って穴埋め学習。
  • PinnerFormer： Pinterest。長期の行動系列から「将来 14 日間の興味」を予測する end-to-end な user embedding。日次バッチで更新する実用設計。

生成型リトリーバル（Generative Retrieval）

「ベクトルを作って ANN で探す」という枠組み自体を捨て、アイテムに Semantic ID（意味的な離散コード列）を割り当て、Transformer で次の Semantic ID 列を 生成することで候補を得る次世代手法。

  • TIGER（Google, 2023）： アイテムの内容埋め込みを RQ-VAE で量子化し階層的な Semantic ID（例: (12, 7, 3)）を付与。系列を入力に次アイテムの ID を生成。意味的に似たアイテムが似た ID 接頭辞を持つため汎化とコールドスタートに強い。
  • P5： 推薦タスク（評価予測・系列推薦・説明生成など）を全部「テキスト to テキスト」に統一し、単一の T5 ベース LM で解く。
▶ なにが嬉しいか
アイテム数に比例した巨大な ANN インデックスを持たず、モデルパラメータに知識を畳み込める。新規アイテムも Semantic ID を振るだけで即座に生成対象になる。研究〜一部本番のフェーズ。

取得を支える基盤技術：ANN（近似最近傍探索）

Two-Tower 等で得た数千万〜億のアイテムベクトルから、クエリベクトルに近いものを 厳密にではなく近似的に高速取得する。これが Retrieval のレイテンシ予算を成立させる心臓部。

手法仕組み特性
HNSW多層のスモールワールドグラフを辿る近傍探索高再現率・低レイテンシ。メモリ消費は大きい。本番デフォルト。
IVFベクトル空間をクラスタ分割し、近いクラスタだけ探索メモリ効率良。nprobe で再現率と速度のトレードオフ。
PQ / OPQベクトルを部分空間に分割し量子化して圧縮メモリを劇的削減。IVF と組み合わせる（IVFPQ）。
BBQ（Better Binary Quantization）ビット量子化＋補正で高圧縮・高速距離計算Elasticsearch/Lucene 系の最新量子化。省メモリで高精度。

実装ライブラリ：FAISSScaNNMilvusHNSWlibVespa。 マルチチャネルの結果はここでマージ・重複除去し、再現率と多様性を両立させてから Pre-ranking へ渡す。

第2段 Pre-ranking（粗選別）

目的：数百〜数千の候補を、重い Ranking に渡す前に数十〜数百まで素早く足切りする「橋渡し」。 「Ranking が重すぎて全候補を捌けない」というコスパ問題を解く段。常に必須ではないが大規模本番では一般的。
観点内容
ルール / ヒューリスティック在庫切れ・既読・年齢/地域/ポリシー制限・NG ワード・スコア閾値などで 機械的に除外。
軽量スコアリングモデルGBDT/XGBoost、浅い MLP、あるいは Two-Tower/Three-Tower で粗い関連度を推定。Ranking モデルの蒸留（distillation）版を使うことも多い。
学習方法多くは二値分類（log loss / BCE）やペアワイズ損失で相対順位の質を担保。
⚠ 一貫性問題（Consistency）
Pre-ranking の役割は「Ranking が高評価する候補を落とさないこと」。よって最適化目標は「絶対精度」より 後段 Ranking との順位の一貫性。近年は Ranking モデルの出力を教師にした knowledge distillation や、両段の順位整合を測る指標で評価する設計が主流。

第3段 Ranking（精密順位付け）

目的：通過した数十〜数百件を、ユーザーの今の状況に合わせて 1 位から超高精度に並び替える。 最もリッチな特徴量（最新価格・在庫・コンテキスト・1 段目のベクトル類似度・人気度…）と最も複雑なモデルを投入する。

進化の流れ：手作り交互作用 → 学習された交互作用 → 系列・グラフモデリング → マルチモーダル / 生成的推論。

ランク学習（Learning to Rank, LTR）

確率の 絶対値ではなく「同じ画面（リスト）内での相対的な上下関係」を学習する枠組み。損失の取り方で 3 流派。

流派考え方代表
Pointwise各アイテムの関連度を独立に回帰/分類回帰、ロジスティック回帰
Pairwise2 アイテムの「どちらが上か」を学習RankNet, LambdaRank
Listwiseリスト全体の順序を直接最適化ListNet, LambdaMART

LambdaMART（LightGBM の lambdarank）は実務の定番。NDCG のような微分不可能な順位指標を、ペアの勾配 $\lambda$ に「その入れ替えで NDCG がどれだけ変わるか」$|\Delta\text{NDCG}|$ を掛けて近似する:

$$\lambda_{ij}=\frac{-1}{1+e^{\,s_i-s_j}}\,\bigl|\Delta\text{NDCG}_{ij}\bigr|$$
▶ クロス特徴量の再利用
1 段目で計算した コサイン類似度・人気度・共起頻度などを、ここでは数値特徴量として GBDT に食わせる。「打順（順位）重視」で、グループ（＝1 回の表示リスト）ごとにランク学習するのがポイント。

クロス特徴モデル（特徴量交互作用の自動学習）

「年齢 × カテゴリ」「都市 × 時間帯」のような特徴量の 掛け合わせ（交互作用）を、人手でなくモデルが学習する。

モデル仕組み
Wide & Deep(Google)暗記担当の線形 Wide 部 ＋ 汎化担当の DNN Deep 部を同時学習。
DeepFMFM（2 次の特徴交差）＋ DNN（高次）を共有埋め込みで結合。手動の特徴エンジニアリング不要。
DCN / DCN v2Cross Network で明示的に「$x_0 x_l^\top w$」と次数を 1 つずつ上げて高次交差を効率的に学習。v2 は低ランク化で実用性↑。

DCN の Cross 層（$l$ 層で $l+1$ 次の交差を生成）:

$$x_{l+1}=x_0\odot(W_l x_l + b_l)+x_l$$

Attention / シーケンスモデル

ユーザーの直近の行動列に Attention をかけ、「今からスコアリングする候補アイテムに関連の深い過去行動」を動的に重み付けする。Taobao・TikTok のフィードで定番。

  • DIN（Deep Interest Network）： 候補アイテムをクエリに、過去行動への Attention で「ユーザーの興味」を候補ごとに動的表現。固定のユーザーベクトルでは捉えられない「多様な興味」を表現。
  • DIEN： DIN に GRU ベースの 興味進化層を追加し、興味の時間変化をモデル化。
  • DSIN： 行動をセッション単位に分割し、セッション内/間の構造を捉える。

マルチタスク学習（MTL）— 実務で超重要

推薦の目的は単一でない。CTR（クリック率）・CVR（転換率）・滞在時間・継続率・いいね…を同時に最適化する。「共有表現＋目的別ヘッド」が基本構造。

モデル仕組み・狙い
MMoE(Multi-gate Mixture-of-Experts)複数の Expert を共有し、タスクごとの Gate が Expert の重みを動的に決定。タスク間の相反（negative transfer）を緩和。
ESMM「CTR」と「CTCVR（クリック→転換）」を同時学習し、CVR のサンプル選択バイアス（クリック後しか転換を観測できない）を構造的に解消。$pCVR = pCTCVR / pCTR$。
PLE(Progressive Layered Extraction)共有 Expert と タスク専用 Expert を分離し層状に抽出。MMoE の seesaw 現象（一方を上げると他方が下がる）を改善。

最終スコアは各目的の予測を重み付き合成するのが典型（重みはビジネス価値で調整）:

$$\text{Score}=pCTR^{\alpha}\cdot pCVR^{\beta}\cdot (\text{price})^{\gamma}\cdot \dots$$

Transformer ベース

長期の系列依存を捉え、テキスト/動画/画像の マルチモーダル特徴も融合する最新潮流。

  • Transformer4Rec： NLP の Transformer 実装（HuggingFace）を系列推薦にブリッジ。
  • P5 / UniRec： 複数の推薦タスクを単一モデルに統合し、自然言語インターフェースで扱う。

第4段 Re-ranking（再順位付け）

目的：Ranking の並びを、関連性・多様性・新鮮さ・ビジネス制約のバランスで最終調整してから表示する。 ここでは「個々のアイテムのスコア」でなく リスト全体・セッション全体の体験を最適化する。

調整したい観点

  • 多様性： 似たアイテムの連続を避ける（アクション映画ばかりにしない）。
  • 公平性 / 露出制御： 出品者・クリエイター・カテゴリ間の露出バランス、露出クォータ。
  • 探索（Exploration）： ロングテール・新着を混ぜ、将来の学習用フィードバックを収集。
  • 新鮮さブースト： 新着・トレンドを一時的に優遇し人気バイアスを是正。
  • セッション満足度： 個別 CTR でなく画面全体・セッション全体を最大化。
レベル手法用途
ルールベースカテゴリ頻度・露出上限など決定論的制約安定・解釈性重視で本番多用
線形/MLP 再スコア元スコア＋新規性/位置バイアス特徴で再採点効率と効果のバランス、業界の中核
コンテキストバンディットLinUCB, Thompson Sampling, EXP3, Neural Bandits活用と探索のバランス（YouTube/TikTok/Taobao）
多様性後処理MMR, DPP（行列式点過程）, MGS冗長性除去の高速ポストプロセス
高度な手法Listwise(ListNet), 強化学習(SlateQ, DQN-Rank), GraphRL計算コスト高く限定/ハイブリッド導入

多様性アルゴリズム：MMR と DPP

MMR（Maximal Marginal Relevance）： 「関連性」と「既選択集合との非類似性」を $\lambda$ で線形にトレードオフし、貪欲に 1 件ずつ選ぶ:

$$\text{MMR}=\arg\max_{i\in R\setminus S}\Bigl[\lambda\,\text{rel}(i)-(1-\lambda)\max_{j\in S}\text{sim}(i,j)\Bigr]$$

DPP（行列式点過程）： 部分集合 $S$ の選ばれやすさをカーネル行列 $L$ の 行列式 $\det(L_S)$ に比例させる。行列式は「ベクトルが張る体積」＝多様性を自然に表現するため、関連性と多様性を確率的に統合できる。

$$P(S)\propto \det(L_S)$$

コンテキストバンディット（探索と活用）

「今わかっている最善（活用）」だけ出すと、新着やニッチが永遠に評価されず学習が止まる。少しだけ「未知への賭け（探索）」を混ぜる枠組み。

  • LinUCB： 各アイテムの報酬期待値の 信頼区間上限が高いものを選ぶ（不確実なものほどボーナス）。
  • Thompson Sampling： 報酬分布の事後分布からサンプリングし、サンプル値が最大のものを選ぶ。実装がシンプルで強力。

LinUCB の選択基準（$\hat\theta$ は推定係数、第 2 項が探索ボーナス）:

$$a_t=\arg\max_a\Bigl(x_a^\top\hat\theta_a+\alpha\sqrt{x_a^\top A_a^{-1}x_a}\Bigr)$$

パイプラインを支える基盤技術スタック

ANN 検索エンジン

FAISS / ScaNN / Milvus / Vespa。HNSW・IVF・PQ・BBQ で億規模ベクトルをミリ秒検索。

Feature Store

Feast / Tecton 等。オンライン（低レイテンシ）/ オフライン（学習用）で同じ特徴量を提供し train-serve skew を防ぐ。

ストリーミング基盤

Kafka / Flink。クリック等の行動をリアルタイムに特徴量へ反映し「直近の興味」を即座に推薦へ。

実験 / 評価

オフライン（NDCG/Recall@K/AUC）＋ オンライン A/B テスト。Interleaving で高速比較も。

⚠ 全段に共通する難所
① フィードバックループ：推薦した結果しかログに残らず、モデルが自分の推薦を強化し続ける。
② 各段の選択バイアス：後段は前段を通過した候補しか学習で見ない。
③ 遅延フィードバック：転換（購入）はクリックの数日後に起きることがあり、ラベルが遅れて確定する。

これからの方向性（次世代）

Unified（統合）

Retrieval・Ranking・生成を単一の LLM フレームワークに統合。段の境界が溶ける。

Multimodal

テキスト・画像・動画・音声を横断的に理解し、コンテンツの「中身」で勝負。

Generative

既存アイテムを「選ぶ」のでなく、パーソナライズされたアイテム/レイアウトを「生成」する。

Privacy-aware

連合学習・オンデバイス学習と LLM 推論の組み合わせで、データを集めずに個別化。
